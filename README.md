# 숨결지기

카메라와 웨어러블 없이 영유아의 수면 상태와 침실 환경을 확인하는 비접촉 모니터링 시스템입니다.
2026 AI소프트웨어공모전 임베디드SW 부문 출품작입니다.

## 기획 의도

저출산 시대의 육아 부담은 단순한 시간 부족을 넘어, 아이 상태를 계속 확인해야 한다는 불안과 수면 부족으로 이어집니다. 특히 신생아와 영유아의 수면 중에는 작은 뒤척임, 울음, 온습도 변화만으로도 부모가 반복해서 깨어 확인하게 됩니다. 기존 베이비 모니터는 카메라 영상이나 웨어러블 착용에 의존하는 경우가 많아 프라이버시 부담과 착용 거부, 충전과 관리 부담이 남습니다.

숨결지기는 아이에게 기기를 붙이지 않고, 아이를 찍지 않으면서도 매일 쓸 수 있는 상태 정보를 제공하는 방향으로 접근했습니다. 60GHz 레이더로 접촉 없이 호흡과 미세 움직임을 추정하고, 마이크로 울음과 제한된 음성 명령을 감지하며, 환경 센서로 온도, 습도, CO2, 조도 같은 침실의 불편 요인을 포착합니다. 이 신호들을 엣지에서 융합하면 아이가 안정적으로 자는지, 뒤척이는지, 깼는지, 울고 있는지, 방 환경이 수면에 적합한지를 하나의 화면으로 보여줄 수 있습니다.

목표는 단일 생체신호의 정밀 진단이 아니라, 여러 센서의 맥락을 결합해 보호자가 이해할 수 있는 상태 판단과 행동 가능한 권고를 제공하는 것입니다. 무호흡 같은 드문 호흡 이상은 상시 모니터링 중 놓치지 않기 위한 안전 백업으로 두었으며, 의료 진단이나 임상 성능을 표방하지 않습니다.

## 구현 기능

1. 비접촉 호흡·움직임 감지: 레이더 신호에서 호흡수, 움직임, 무호흡 의심을 추정합니다.
2. 수면/각성 상태 분류: 안정 수면, 뒤척임, 각성을 구분해 일상 모니터링의 기본 정보로 제공합니다.
3. 울음 감지와 환경·시간 맥락 분석: 울음의 이유를 단정하는 대신, 울음이 어떤 환경 조건이나 시간대와 함께 반복되는지를 분석해 참고 정보로 제공합니다.
4. 개인화 정상 범위: 최근 관찰된 호흡수와 움직임을 누적해 아이별 정상 범위를 점진적으로 갱신합니다.
5. 환경 모니터링과 케어 권고: 습도가 낮으면 가습을, 수면 중 조도가 높으면 조명 낮춤을 권고하고, CO2는 환기 필요성을 알리는 보조 정보로 사용합니다.
6. 멀티모달 융합 판단: 레이더, 울음, 환경, 수면 상태, 개인화 기준을 함께 고려해 위험도를 판단하고, 순간적인 센서 튐은 연속 확인 과정을 거쳐 걸러냅니다.
7. 알림 안정화: 같은 상태가 짧은 시간 동안 확인될 때만 알림을 확정하고, 같은 유형의 알림이 반복되지 않도록 억제합니다.
8. 수면 리듬 분석과 아침 리포트: 밤새 누적된 상태를 요약해 울음 횟수, 자주 깬 시간대, 안정 수면 비율 등을 아침 리포트로 제공합니다.
9. 부모 상호작용과 대시보드: 현재 상태와 환경, 권고, 이벤트를 대시보드로 보여주고 제한된 음성·터치 입력을 지원합니다.

## 시스템 구조

레이더, 마이크, 환경 센서 입력이 각각 엣지에서 처리된 뒤 융합 판단으로 모이고, 그 결과가 대시보드와 알림, 환경 권고, 아침 리포트로 나갑니다. 모든 모듈은 정해진 메시지 주제(토픽) 체계로만 통신하기 때문에, 검증에 사용한 가상 센서를 실물 센서로 바꾸더라도 상위 판단 로직과 화면 구조를 그대로 유지할 수 있습니다.

기본 실행은 단일 프로세스 안에서 pub/sub로 동작합니다. 환경변수 `NUNI_BUS=mqtt`로 전환하면 MQTT 브로커를 통한 분산 모드로 같은 코드가 동작합니다.

주요 모듈은 다음과 같습니다.

- `radar_dsp.py`, `radar_sim_signal.py`: 레이더 신호 처리와 합성 신호 생성
- `cry_classifier.py`, `cry_context.py`: 울음 감지와 환경·시간 맥락 분석
- `fusion.py`: 멀티모달 융합 판단과 디바운스·쿨다운
- `sleep_state.py`, `sleep_rhythm.py`: 수면/각성 분류와 수면 리듬 요약
- `personalization.py`: 개인 baseline 온라인 학습
- `env_control.py`: 환경 케어 권고
- `voice_intent.py`: 제한된 음성 인텐트 인식
- `cloud.py`: 비실시간 아침 수면 리포트
- `app.py`: Streamlit 대시보드
- `bus.py`, `topics.py`, `state_store.py`: 메시지 버스와 상태 저장

## 검증 결과

현재 정량 결과는 합성 신호와 통제된 시나리오를 이용한 소프트웨어 검증값입니다. 실제 60GHz 하드웨어나 실제 영유아, 임상 환경의 성능이 아닙니다.

| 항목 | 결과 |
|---|---|
| 호흡수 추정 | 합성 신호 기준 평균 절대 오차 약 0.5회/분 |
| 수면/각성 분류 | 통제된 7개 시나리오에서 기대값과 일치 |
| 개인화 baseline | 평균 40, 52회/분인 두 가구 데이터가 각각 40.1, 51.7 부근으로 수렴 |
| 환경 권고 | 통제된 8개 시나리오에서 기대값과 일치 |
| 멀티모달 융합 | 통제된 11개 시나리오에서 융합 판단 오탐·미탐 0 |
| 융합 강건성 | 무작위 잡음 2000회 반복에서 단일 센서 오탐률 57.6%, 융합 2.7% |
| 알림 안정화 | 스트리밍 검증에서 알림 4건에서 1건, 일시 오경보 3건에서 0건 |
| 수면 리듬 | 합성 5박 데이터에서 안정 수면 87%, 자주 깬 시간대 확인 |

`run_all.sh`를 실행하면 위 항목을 순서대로 재검증하고 결과 파일을 다시 생성합니다.

## 실행 방법

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

bash run_all.sh        # 검증 파이프라인 실행
streamlit run app.py   # 대시보드 실행
```

대시보드 사이드바에서 울음이나 무호흡 시나리오를 주입하면 융합 판단이 어떻게 반응하는지 확인할 수 있습니다.

## 2차 데모 계획

2차 데모의 목표는 새 기능을 다시 만드는 것이 아니라, 지금까지 검증한 소프트웨어를 실물 센서와 화면에 연결하는 것입니다. 기존 메시지 주제 체계와 데이터 형식을 유지하면 융합 판단, 상태 저장, 대시보드, 리포트, 개인화, 환경 권고 기능은 큰 변경 없이 재사용할 수 있습니다.

- 가상 입력을 실제 60GHz 레이더, 환경 센서, 마이크 입력으로 교체합니다.
- 성인 대상 비접촉 호흡 측정으로 실물 레이더 입력에서 호흡 신호가 추출되는지 확인합니다. 이는 실물 센서와 소프트웨어 처리의 연결을 확인하기 위한 기능 검증이며, 영유아 대상 실험이나 의료적 검증이 아닙니다.
- 가습기, 조명 또는 모의 액추에이터를 연결해 권고가 실제 동작으로 이어지는지 확인합니다. 안전을 위해 초기에는 보호자 확인 후 실행되는 반자동 방식을 우선 적용합니다.
- 인형 기반 모의 환경에서 습도·조도 변화와 울음·움직임 이벤트를 주입해 환경과 상태 패턴을 함께 기록하는 흐름을 확인합니다.

## 한계

- 무호흡과 호흡 이상 감지는 드문 상황을 위한 안전 백업이며, 의료기기나 진단을 표방하지 않습니다.
- 울음 이유 분류는 공개 데이터의 불균형 때문에 실험적 기능으로 두고, 대신 울음과 환경·시간의 상관을 참고 정보로 제공합니다.
